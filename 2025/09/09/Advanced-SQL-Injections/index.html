Introduction to PostgreSQL

引言

在本模块中，我们将探讨一些 advanced SQL injection 技术以及从 white-box approach 角度出发的 PostgreSQL 特定攻击。由于这是一个高级模块，期望读者具备 SQL 语法、SQL 注入和 Python3 的基础知识，以便全面理解所解释的概念。尽管本模块将重点放在 PostgreSQL 上，但同样的技术可以适应于其他 SQL 变体，因为 SQL 是一种标准化的语言。

与 PostgreSQL 交互

在深入了解注入漏洞之前，我们先花点时间熟悉一下与 PostgreSQL 数据库交互的两个最常用工具：psql 和 pgAdmin4。

psql (PostgreSQL Interactive Terminal)

psql 是一个与 PostgreSQL 数据库交互的 command-line 工具，它随 PostgreSQL 服务器预打包，并可在 Linux 或 Windows 上运行。

安装完成后，您可以从安装目录（默认为 C:\Program Files\PostgreSQL\15\bin ）使用 psql.exe ，或者将该目录添加到系统 PATH 变量中，以便从任何地方使用它：

pgAdmin4

pgAdmin4 是一个用于与 PostgreSQL 数据库交互的 GUI 应用程序，适用于 Linux 和 Windows 平台。

要在 Windows 上安装 pgAdmin4 ，您可以从 pgadmin.org 下载安装程序并按照安装步骤进行，或者您可以重复使用我们安装 psql.exe 时所用的安装程序，只需确保在安装过程中勾选 pgAdmin4 选项即可。

首次启动 pgAdmin4 时，您需要设置一个主密码。这主要是为了保护您后续用于连接数据库的凭证。

要连接到 PostgreSQL 服务器，请前往 Dashboard > Add New Server ，填写详细信息并按下 Save 。

完成此操作后，您可以在所选名称下的 left-hand side 访问 server 。通过此图形界面查看数据库和表非常直观，运行查询就像在 database 上 right-clicking 并选择 Query Tool 一样简单。
